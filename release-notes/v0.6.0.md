# x-agentic-workflow v0.6.0

This release is a Provider Settings and desktop safety milestone after the
v0.5 macOS preview distribution.

The public release pattern we are following is: ship small desktop/provider
increments with explicit tested commands, quality evidence, confidence, and
scope risk. The implementation remains XAW clean-room code.

## Highlights

- **Desktop Provider Settings**: the local browser UI now exposes provider,
  model, base URL, and API-key environment variable controls.
- **Metadata-only persistence**: saving provider settings writes provider
  metadata to the XAW config file but does not store API key values.
- **Connectivity check**: the settings panel can run a minimal provider
  connection test through the same provider abstraction used by the runtime.
- **Redacted provider errors**: connection-test failures redact configured API
  key values and token-like strings before displaying messages in the browser.
- **Desktop shell cleanup**: the Codex-style desktop shell keeps the corrected
  `354685856-sn/x-agentic-workflow` project label and removes accidental
  workspace-note text from visible UI.

## Fixes

- Removed accidental `我的仓库位置...` text from the desktop sidebar and tab.
- Added validation for unsupported provider names, empty model values, and empty
  API-key environment variable names in provider connection tests.
- Added regression tests so secret-like values do not leak through provider test
  error rendering.

## Confidence

Medium-high for the desktop Provider Settings surface and redaction behavior.

## Scope Risk

Moderate. The browser desktop UI changed substantially, but the runtime provider
abstraction, CLI entrypoints, sessions, tools, and package distribution scripts
remain scoped out of this feature except for version/documentation alignment.

## Not Included

- Keychain-backed secret storage.
- Full provider preset management.
- Real provider smoke with a live API key.
- Production signed/notarized macOS distribution.
