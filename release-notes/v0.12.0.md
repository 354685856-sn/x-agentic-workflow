# x-agentic-workflow v0.12.0

This release adds real desktop session recovery cues and local session filtering
without introducing placeholder plugin or scheduled-task navigation.

## Highlights

- **Session summaries**: desktop sessions now expose title, update timestamp,
  message count, and file-change count.
- **Restored-session cues**: opening a previous session updates the main title
  and shows a visible restored-session pill.
- **Searchable session list**: the sidebar can filter local sessions by session
  title or id.
- **Compatibility preserved**: `SessionStore.load()` remains message-focused for
  existing CLI/TUI callers.

## Scope Risk

Moderate. The change extends session metadata reads and desktop rendering, but
does not change provider calls, tool execution, command approval, or persisted
message loading behavior.
