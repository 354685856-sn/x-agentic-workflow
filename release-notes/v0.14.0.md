# x-agentic-workflow v0.14.0

This release makes the desktop Provider Settings screen reflect real saved
configuration and hardens save/test interactions against draft loss and
duplicate submissions.

## Highlights

- **Real saved-provider card**: displays the current provider protocol, model,
  base URL, API key environment variable, and whether that environment variable
  is available.
- **Draft preservation**: unsaved form values survive chat/settings transitions
  and unrelated desktop state refreshes.
- **Visible status**: settings distinguishes saved configuration, missing API
  key environment, and unsaved changes.
- **Duplicate-submit guard**: save and test actions are disabled while a provider
  request is active.
- **No fake provider controls**: removes the inactive add-provider button and
  hardcoded sample provider cards.

## Confidence

Medium-high for the scoped Provider Settings UX.

## Scope Risk

Low-to-moderate. The provider APIs and secret-storage policy are unchanged; the
release updates form state, request coordination, and saved-state rendering.

## Not Included

- Multiple simultaneously saved providers.
- Provider deletion, ordering, or import.
- Keychain secret storage.
- PyPI upload or DMG rebuild.
