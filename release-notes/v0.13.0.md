# x-agentic-workflow v0.13.0

This release turns the desktop composer `+` into a real, bounded text-file
attachment flow. It deliberately does not expose placeholder Skills,
Connectors, Plugins, image, or PDF actions.

## Highlights

- **Real composer attachment action**: `+` opens the native picker for supported
  text/code files.
- **Attachment chips**: selected files appear as removable chips before send.
- **Bounded context**: maximum five files, 128 KiB per file, and 256 KiB total.
- **Server-side validation**: filenames, content type shape, and byte limits are
  checked again by the desktop backend.
- **Stale-read protection**: sends, session changes, new chats, and project
  switches invalidate unfinished asynchronous file reads.
- **Clean session restore**: restored transcripts show the prompt and attachment
  names without rendering raw attachment bodies in the conversation UI.

## Confidence

Medium-high for the scoped text-attachment flow.

## Scope Risk

Moderate. Attachments add user-controlled prompt context and asynchronous
browser file reads, both bounded and covered by client/server validation.

## Not Included

- Images, PDFs, or binary attachments.
- Drag-and-drop or `@mention`.
- Skills, Connectors, or Plugins menus.
- PyPI upload or DMG rebuild.
