# x-agentic-workflow v0.8.0

This release continues the desktop Project/workspace milestone by letting the
local browser UI switch the active project path after validating it.

## Highlights

- **Project switching**: the desktop composer now includes a project path input
  and `切换项目` action.
- **Recent projects**: switched project paths are persisted in local config and
  rendered in the sidebar as recent projects.
- **Automatic validation**: after a successful switch, the desktop runtime
  resets the chat state, rebuilds the agent for the new workdir, and re-runs
  the read-only project validation report.
- **Invalid path feedback**: nonexistent paths and non-directory paths are
  rejected without changing the active project.
- **Cleaner sidebar**: static demo project rows were replaced with dynamic
  current-project and recent-project rows.

## Confidence

Medium-high for the scoped project-switching slice.

## Scope Risk

Low-to-moderate. The feature changes desktop workdir state and persists recent
project paths, but it does not run project commands automatically or introduce
per-project session storage yet.

## Not Included

- Per-project session list.
- File-change ledger.
- Desktop diff workspace.
- Native folder picker.
- Automatic execution of recommended validation commands.
