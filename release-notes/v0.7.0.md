# x-agentic-workflow v0.7.0

This release starts the Project/workspace milestone with a small, verifiable
desktop workflow: validate the current project before asking the agent to work.

## Highlights

- **Validate Project button**: the desktop composer now includes a `验证项目`
  action with a disabled `验证中...` state while the check is running.
- **Read-only project health check**: validation reports whether the current
  project path exists, which key files are present, and the current git state.
- **Current project label**: the desktop shell now renders the project label
  from the active workdir instead of shipping a copied demo project name.
- **Suggested verification commands**: Python projects receive recommended
  `pytest`, `ruff`, and `mypy` commands; JavaScript projects receive common
  `npm` checks.
- **Inspector report**: validation results render in the right inspector so the
  user can see project readiness without switching views.

## Tested

## Confidence

Medium-high for the read-only project validation slice.

## Scope Risk

Low-to-moderate. The feature adds a new desktop API route and UI rendering, but
does not execute project tests, mutate files, or switch workdirs.

## Not Included

- Multi-project selector.
- Workdir switching.
- Per-project sessions.
- Running validation commands automatically.
- File-change ledger or desktop diff workspace.
